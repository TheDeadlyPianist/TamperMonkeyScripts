Auth Login Helper - URL Helper

Why use the helper?

URLs can be long and complicated and tedious to remember.
Often you need to construct them (especially locally) from multiple parts and it can be annoying and take up time.

What if you had a tool that would construct the entire URL for you, and even adjust the output based on whether it is on the local Auth Login or one in the environments?

Well, that is exactly what this tool will help you do.
Gone are the days of writing out or pasting in the same URL over and over during presentations or showcases. You can now quickly and easily store URLs in an organised fashion and select them as and when needed.

Features

  • Easily select stored URLs
  • Delete unwanted URLs with a click of a button
  • Create groups to contain different list that share a common theme
  • Create list to store URLs with a common route
  • Create individual URLs
  • The URL produced is formatted environment appropriate.
  • You can hover over a list to get a small popup to remind you what the base URL for said list is.

How to Use

Create a new URL

Three buttons are create above the Redirect Url section:

  • New URL Group
  • New Base URL
  • New Redirect URL

Create URL Group

A URL group is just a collection or URL list with a common theme. E.g. The group could be the team, and the list could be individual micro services.

Step 1

Clicking on the New URL Group button will give you a new input box.
Pressing submit will create the new category with the given name.
Pressing cancel will close the input.




Step 2

Type in a name, then click submit. This will generate a box that is ready to contain some list.




Create base URL

A base URL is a root URL common to all child URLs. This exist so you don't need to type out the same base URL every time.
E.g.

  • http://somewebsite/page/apage
  • http://somewebsite/page/asecondpage
These links both contain a common "http://somewebsite/page/", and thus can be treated as a base URL.

Step 1

When pressing "New Base URL", you will get 2 input boxes followed by a drop down.
The first box is what you want to call the Base URL. This is only for the frontend.
The second box is where the base url goes.




Step 2

The drop down box will contain a list of all groups that are available. This is where the base URL will be placed.

Create new Redirect URL

A redirect URL is the unique part of each url. From the above example, these would be:

  • apage
  • asecondpage
Step 1

Pressing "New Redirect URL" will open 2 inputs and 2 dropdowns. The inputs work in very much the same way as a new base URL.
The first dropdown once again refers to the group you want it placed in, and the second dropdown will be what base URL within that group you want the redirect URL to be placed under.
